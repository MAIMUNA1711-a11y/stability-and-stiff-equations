Concept 01
What Makes an ODE Stiff?

Stiffness occurs when an ODE contains timescales that differ by orders of magnitude. The fast component dies out quickly but forces a tiny step size on explicit methods — even when we only care about the slow component. All stability theory is developed using dy/dt = λy. This test equation captures the essential behavior of stiff systems through its eigenvalue λ.

FOUNDATION — THE TEST EQUATION
dy/dt = λy  →  y(t) = y₀ · eλt
This simple test equation is the foundation of all stability analysis. Its exact solution is y(t) = y₀ · eλt.

When Re(λ) < 0 — the solution decays exponentially toward zero. A good numerical method must preserve this decay.
Re(λ) < 0  →  y(t) → 0 as t → ∞  (decays ✓)
Re(λ) > 0  →  y(t) → ∞ as t → ∞  (unstable ✗)
Re(λ) = 0  →  oscillatory, marginally stable
👉 The stiff example below uses Slide 15's equation, where λ₁ = −1000 (fast) and λ₂ = −1 (slow) create a stiffness ratio of 1000 — forcing the numerical method to respect h < 2/|λ_max|.
KEY INSIGHT
Stiffness is NOT about fast change alone. It is about the stability restrictions imposed by fast-decaying components, even when the actual solution of interest changes slowly. The numerical method is forced to take tiny steps — not for accuracy, but purely for stability.

The Stiff ODE Example

dy/dt = −1000y + 1000e⁻ᵗ

Exact solution: y(t) = e⁻ᵗ − e⁻¹⁰⁰⁰ᵗ  (with y(0) = 0)

Two timescales: λ₁ = −1000 (fast) and λ₂ = −1 (slow). The fast term e⁻¹⁰⁰⁰ᵗ vanishes almost instantly, but any explicit method must step as if it persists forever.

Fast component: −1000y  (decays very quickly, eigenvalue λ₁ ≈ −1000)
Slow component: 1000e⁻ᵗ  (changes slowly, eigenvalue λ₂ ≈ −1)
Stiffness ratio ≈ 1000 → stiff problem requiring implicit methods.

Stiffness Ratio

S = |λ_max / λ_min| = 1000/1 = 1000

The stiffness ratio measures how separated the eigenvalues are. For Forward Euler, stability requires h < 2/|λ_max| = 0.002.

To simulate to t = 1, we need ≥ 500 steps — just to track a solution that changes slowly!

Concept 02
The Amplification Factor

For the test equation dy/dt = λy, the exact solution decays as eλt (Re(λ)<0). The amplification factor R(z) tells us how much each numerical step multiplies the solution. Stability demands |R(z)| < 1 at every step — so that the numerical solution also decays, matching the exact behavior. This single number predicts stability or explosion.

Forward Euler

R(z) = 1 + z

where z = hλ. After n steps, yₙ = R(z)ⁿ · y₀.

Stability ⟺ |R(z)| = |1 + hλ| < 1
This traces a circle of radius 1 centered at z = −1.

Backward Euler

R(z) = 1/(1 − z)

For Re(z) < 0, we have |R(z)| < 1 always.

The entire left half-plane is stable — any step size works for any λ with Re(λ) < 0.
This is A-stability.

BE: R(z) = 1/(1−(−5)) = 1/6 ≈ 0.167
|R(z)| = 0.167 < 1  → STABLE ✓
(vs FE: |R|=4 → UNSTABLE at same h)

Concept 03
Stability Regions in the Complex Plane

The stability region is the set of all z = hλ values for which |R(z)| ≤ 1. The shape of this region determines which problems each method can handle safely.

A method is A-stable if its stability region contains the entire left half-plane {z : Re(z) ≤ 0}. This guarantees stability for any step size on any problem with decaying eigenvalues.

⚑ Stability theory uses dy/dt = λy as the test equation. The stiffness comes from eigenvalues with large negative real parts (Re(λ) ≪ 0), forcing tiny step sizes on explicit methods — this is the stability restriction, not an accuracy requirement.

Concept 04

Forward Euler is simple but fragile. When h is too large relative to λ, the amplification factor |1 + hλ| exceeds 1 — and errors grow exponentially instead of decaying.

Forward Euler:  yₙ₊₁ = yₙ + h·f(tₙ, yₙ)   ⟹   yₙ = (1 + hλ)ⁿ · y₀
Stable only when h < 2/|λ|. Critical threshold is a hard boundary.

Concept 05
Backward Euler — A-Stability

Backward Euler evaluates the derivative at the next time point (implicitly). This simple change makes it unconditionally stable for any step size — but requires solving an equation at each step. Applied to the test equation dy/dt = λy, it gives R(z) = 1/(1−z), which satisfies |R(z)| < 1 for all Re(z) < 0 regardless of h.

Forward Euler:  yₙ₊₁ = yₙ + h·f(tₙ, yₙ)
↳ Explicit: use current values
Backward Euler:  yₙ₊₁ = yₙ + h·f(tₙ₊₁, yₙ₊₁)
↳ Implicit: solve for unknown yₙ₊₁
For dy/dt = λy:    yₙ₊₁ = yₙ/(1 − hλ)   ⟹   |R(z)| = |1/(1−z)|
For Re(λ) < 0 and any h > 0: |1/(1−hλ)| < 1 always. ∴ A-stable ✓

Why Implicit = A-stable?

R(z) = 1/(1 − z). For z = hλ with λ = −a + bi (a > 0): |1 − z|² = (1 + ah)² + (bh)² > 1. So |R(z)| = 1/|1−z| < 1 always.

No matter how large h is, the implicit method damps the solution. It may be inaccurate, but it will never blow up.

Concept 06
RK2 vs RK4 — Stability Comparison

Runge-Kutta methods use multiple stage evaluations to achieve higher accuracy. Higher order also typically means a larger stability region — but they remain explicit, so they cannot be A-stable.

RK2 — Midpoint Method

R(z) = 1 + z + z²/2

Uses 2 stage evaluations. Real-axis stability limit: |hλ| ≤ 2 — same as Forward Euler on the real axis, but imaginary axis coverage is better.

RK4 — Classical Method

R(z) = 1 + z + z²/2 + z³/6 + z⁴/24

Uses 4 stage evaluations. Real-axis stability limit: |hλ| ≤ 2.79. For accuracy on smooth problems, RK4 is the workhorse of numerical ODE solving.

Conceptual Summary

The fundamental insight: stiff ODEs force a trade-off between stability and computational cost.


Explicit methods (FE, RK4) compute derivatives at known points — cheap, but require tiny steps for stiff problems.


Implicit methods (BE, BDF) solve a system at each step — expensive, but unconditionally stable, ideal for stiff ODEs.


The stability region in the complex plane is the geometric answer to: "Which values of hλ are safe?"

BOTH INTERPRETATIONS Test equation dy/dt = λy → solution decays to 0. Used for all stability analysis and theory development.

Real stiff systems (e.g. dy/dt = −1000y + 1000e−t) → fast eigenvalue λ₁ = −1000 forces tiny step sizes on explicit methods.

The stability restriction is always governed by the largest |Re(λ)|. Stiffness is about the step-size penalty imposed by λ_max.

Limitations & Improvements
What Goes Wrong — and How to Fix It

Every numerical method has boundaries. Understanding where they fail — and which alternatives succeed — is the core practical lesson of stability theory.

⚠ Limitations
❌
Forward Euler becomes unstable for large h

Stability requires |1 + hλ| < 1 → h < 2/|λ|. For stiff systems with |λ| ≫ 1, this forces an extremely small step size, making the method computationally expensive even when the solution changes slowly.

❌
All explicit methods fail for stiff equations

Explicit methods (FE, RK2, RK4) have finite stability regions. For stiff ODEs where eigenvalues span several orders of magnitude, no explicit method can take a step size large enough to be efficient.

❌
Stability ≠ Accuracy

A method can be stable (values don't blow up) but still inaccurate. Example: FE with z = −1 gives |R| = 0 — the solution collapses to zero in one step, giving 100% error despite being "stable".

✅ Solutions & Improvements
🛡️
Use Backward Euler (A-stable)

Backward Euler's stability region is the entire left half of the complex plane. It is unconditionally stable for any h when Re(λ) < 0 — the standard case for physical systems that decay.

yₙ₊₁ = yₙ / (1 − hλ)
⚙️
Use implicit methods for stiff problems

General implicit methods (Crank-Nicolson, BDF, SDIRK) solve a system of equations at each step but can take arbitrarily large steps for stiff problems. Libraries like SciPy's solve_ivp use them automatically.

📐
Reduce step size h

The simplest fix: keep h < 2/|λ_max|. Not efficient for stiff problems, but valid for mildly stiff or moderate λ values. Adaptive step-size control (e.g., RK45) does this automatically.

🔬
Use higher-order implicit methods

RK4 has a larger stability region than FE or RK2 and achieves O(h⁴) accuracy. For very stiff problems, BDF (Backward Differentiation Formulas) methods offer both A-stability and high accuracy.

🐍 Python Code Example — Forward Euler vs Backward Euler
fe_vs_be.py 
# Forward Euler vs Backward Euler for dy/dt = λy
import numpy as np
import matplotlib.pyplot as plt

lam, h, y0, n = -50, 0.1, 1.0, 10

# Forward Euler: yₙ₊₁ = yₙ(1 + hλ)  — explicit
t_fe, y_fe = [0], [y0]
for _ in range(n):
    y_fe.append(y_fe[-1] * (1 + h * lam))
    t_fe.append(t_fe[-1] + h)

# Backward Euler: yₙ₊₁ = yₙ/(1 − hλ)  — implicit, always stable
t_be, y_be = [0], [y0]
for _ in range(n):
    y_be.append(y_be[-1] / (1 - h * lam))
    t_be.append(t_be[-1] + h)

# Exact solution
t_ex = np.linspace(0, n*h, 300)
y_ex = y0 * np.exp(lam * t_ex)

# Print stability info
print(f"z = hλ = {h*lam}")
print(f"|R_FE| = {abs(1+h*lam):.2f}  → {'STABLE' if abs(1+h*lam)<1 else 'UNSTABLE'}")
print(f"|R_BE| = {abs(1/(1-h*lam)):.4f} → STABLE (always)")

plt.plot(t_ex, y_ex, '--', label='Exact')
plt.plot(t_fe, y_fe, 'o-', label='Forward Euler ✗', color='red')
plt.plot(t_be, y_be, 's-', label='Backward Euler ✓', color='cyan')
plt.legend(); plt.show()
